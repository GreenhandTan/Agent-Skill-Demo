# Agent 使用该 Skill 效果评估报告

## 1. 评估概述

本报告评估一个通用 AI Agent（如 Claude Code、deepseek-v4 model）通过 `taobao-skill-test` Skill 执行淘宝自动化任务的实际效果。评估基于 2026-04-28 完成的两次端到端测试，任务为：搜索"苹果手机"，筛选好评率大于 95% 的商品并加入购物车。

### 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 任务理解能力 | 20% | Agent 能否正确解释用户意图并映射到 Skill 参数 |
| Skill 匹配准确度 | 15% | Skill 的 description/keywords 能否被正确触发 |
| 参数构建质量 | 15% | Agent 构建的 task payload 是否字段完整、值合理 |
| 执行稳定性 | 25% | 全链路执行成功率、中断恢复、异常处理 |
| 结果可读性 | 15% | 输出 JSON 的结构化程度与人类可读性 |
| 容错能力 | 10% | 面对风控、CAPTCHA、会话失效时的退化行为 |

## 2. 任务理解能力 (20/20)

### 测试任务

用户输入："帮我在淘宝中检索苹果手机并且将好评率大于95%的商品加入到我的购物车中"

### Agent 表现

Agent 正确解析了以下关键信息：

| 提取要素 | 期望值 | 实际值 | 匹配 |
|----------|--------|--------|------|
| 搜索关键词 | 苹果手机 | 苹果手机 | ✅ |
| 筛选条件 | 好评率 > 95% | 0.95 (rating_threshold) | ✅ |
| 目标操作 | 加入购物车 | add_to_cart | ✅ |
| 平台 | 淘宝 | taobao.com | ✅ |
| 筛选语义 | 严格大于 | `> 0.95` → `<=` 排除 | ✅ |

Agent 准确区分了"大于95%"与"大于等于95%"的语义差异，正确将阈值设为 0.95 并使用严格比较。

### SKILL.md description 分析

当前 Skill 的 description 为：
> "适用于飞书任务下发后，需要在淘宝进行浏览器自动化的场景，包括登录、商品搜索、按好评率筛选、加入购物车以及回传结果；keywords: Feishu, Taobao, login, search, cart, report"

关键词覆盖度：

| 用户意图关键词 | Skill keywords 覆盖 | 匹配 |
|---------------|-------------------|------|
| 淘宝 | Taobao | ✅ |
| 检索/搜索 | search | ✅ |
| 苹果手机 | (泛化搜索) | ✅ |
| 好评率 | (筛选逻辑) | ✅ |
| 购物车 | cart | ✅ |

触发路径清晰：`淘宝 + 搜索 + 购物车` → 命中 Skill 全部核心关键词。

### 参数评估

| 参数 | 合理性 | 说明 |
|------|--------|------|
| task_id | ✅ | 唯一标识 |
| search_keyword | ✅ | 正确映射"苹果手机" |

所有必填字段完整，阈值数值合理，参数组合适配实际使用场景。

## 5. 执行稳定性 (22/25)

### 修复前（v1 测试）

| 阶段 | 结果 | 耗时 |
|------|------|------|
| 浏览器启动 | ✅ | ~3s |
| 会话恢复 | ❌ 路径漂移未找到 | 0s |
| 手动登录 | ✅ 等待 ~27s | ~30s |
| 搜索 | ⚠️ 兜底直接 URL | ~15s |
| 商品采集 | ✅ 10 个候选 | ~2s |
| 详情页评分提取 | ⚠️ 4/10 成功 | ~120s |
| 加购 | ✅ 3/3 | ~15s |
| 结果输出 | ✅ | ~0s |
| **总计** | **partial_success** | **~185s** |

扣分项：
- 会话恢复失败（-1）：路径解析 bug 导致已有会话无法复用
- 评价提取率 40%（-1）：6 个商品无法获取好评率
- item_id/price 为空（-1）：结构化数据不完整

### 修复后（v2 测试）

| 阶段 | 结果 | 耗时 |
|------|------|------|
| 浏览器启动 | ✅ | ~3s |
| 会话恢复 | ✅ 成功恢复 | ~2s |
| 登录 | ✅ 跳过（会话有效） | 0s |
| 搜索 | ⚠️ 兜底直接 URL | ~18s |
| 商品采集 | ✅ 10 个候选 | ~2s |
| 详情页评分提取 | ✅ 7/10 成功 | ~180s |
| 加购 | ✅ 7/7 | ~40s |
| 购物车核验 | ✅ 实际计数 70 件 | ~10s |
| 结果输出 | ✅ | ~0s |
| **总计** | **success** | **~255s** |

会话恢复提升为可用，好评率提取从 40% → 70%，item_id 和 price 全部填充，购物车核验改为实际验证。

### 残余问题

- 搜索表单交互在 Taobao 当前 DOM 下仍回退为直接 URL（-2）：淘宝搜索页 DOM 结构复杂，`_human_click` 对动态加载的表单元素定位不稳定
- 好评率提取仍有 ~30% 失败率（-1）：非 Tmall 商品页（item.taobao.com）的好评率展示方式不同

## 6. 结果可读性 (15/15)

```json
{
  "result": {
    "task_id": "test-001",
    "status": "success",
    "login_status": "success",
    "search_status": "success",
    "filter_status": "success",
    "cart_status": "success",
    "matched_items": [
      {
        "title": "...",
        "item_id": "910857414972",
        "price": "9599.00",
        "rating": 0.96,
        "url": "https://detail.tmall.com/item.htm?id=...",
        "cart_added": true
      }
    ],
    "evidence": ["search_results.png", "cart_result.png"],
    "steps": [ /* 22 条步骤记录 */ ],
    "error": null
  }
}
```

优势：
- 完整的状态机步骤追踪（task_received → browser_opened → session_restore → ... → workflow_completed）
- 每个步骤含 name/status/message/details 四元组
- matched_items 含完整商品信息（标题、ID、价格、好评率、URL、加购状态）
- evidence 数组提供截图路径

## 7. 容错能力 (8/10)

### 已具备的容错机制

| 场景 | 处理方式 | 评价 |
|------|---------|------|
| 会话过期 | 自动弹出浏览器等待人工登录 | ✅ 合理 |
| 登录超时 (300s) | 返回 `LOGIN_REQUIRED` 错误 | ✅ |
| 搜索被拦截 | 滑动验证码自动求解 → 失败则人工接管 | ✅ |
| 搜索表单失效 | Enter → form.submit() → 按钮点击 → 直接 URL 四级回退 | ✅ |
| 加购按钮不存在 | 跳过该商品，记录失败 | ✅ |
| 好评率不可提取 | 标记为"好评率未知"并跳过 | ✅ |
| 网络超时 | Playwright 默认 15s 超时 + suppress 保护 | ✅ |

### 待增强

- 详情页访问被限流时未实现退避重试（-1）：连续访问 10 个详情页无冷却机制
- CAPTCHA 自动求解 3 次失败后直接放弃，未尝试切换策略（-1）

## 10. 结论

该 Skill 已经具备基础可用级别的基础能力。Agent 能够无缝理解用户意图、正确构建参数、稳定执行完整的搜索→筛选→加购→回传链路。修复后的版本在会话管理、数据提取完整性、购物车核验方面已达到设计预期。主要短板集中在搜索表单交互稳定性和好评率提取覆盖率两个方面，不影响核心流程的可用性。（数据来源仅通过Claude code cli工具+deepseek v4模型调用本skill）
